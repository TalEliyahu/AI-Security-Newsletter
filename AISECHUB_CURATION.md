# AISecHub Curation Profile

Use this profile when selecting posts for the AISecHub Telegram channel.

## Observed Channel Pattern

- Post format is usually one item per Telegram message.
- Default style is short: title on the first line, URL on the second line.
- Posts should not mention automation, bots, testing, or the posting workflow.
- Prefer primary sources over reposts: vendor security research, engineering blogs, arXiv papers, official advisories, GitHub repos, and technical reports.
- Short bursts of several high-signal links are acceptable.

## Preferred Topics

Highest priority:

- AI agent security, especially tool use, runtime controls, and autonomous cyber capability.
- MCP security, including tool poisoning, server vulnerabilities, context injection, auth, and supply chain risks.
- Prompt injection, indirect prompt injection, hidden instructions, and agentic browser attacks.
- AI coding-agent and skill/plugin supply chain security.
- AI red teaming, evals, cyber benchmarks, and exploit-generation capability measurement.
- Technical AI security tools, datasets, frameworks, labs, and GitHub repositories.
- Security automation and AI SOC workflows when implementation details are present.

Medium priority:

- Official AI security governance, standards, and risk frameworks when operationally useful.
- High-quality vulnerability research that is not AI-specific but relevant to offensive capability, exploit automation, or security engineering.

Reject or heavily down-rank:

- Generic AI news, funding, product marketing, opinion-only posts, and SEO-style explainers.
- Items with no technical substance, no primary source, or unclear provenance.
- Duplicates already posted to the channel.
- Paywalled links unless the title/source is unusually important.

## Daily Posting Workflow

1. Read recent channel history from `https://t.me/s/AISecHub` and avoid duplicate URLs/titles.
2. Search current primary sources for new AI security content.
3. Select 3-5 items using this ranking:
   - concrete exploit or vulnerability mechanics
   - primary-source credibility
   - relevance to agent/MCP/prompt-injection/supply-chain security
   - novelty and recency
   - usefulness to practitioners and researchers
4. Post each item separately in title+URL format.
5. Verify the latest public channel preview includes the new messages.
